# Design reference: dgcp.gob.do flow (Kelvin's pick, 2026-06-10)

Walked live 2026-06-10; screenshots ~/dgcp-top.png, ~/dgcp-mid1-3.png.

What makes the DGCP layout work (the parts to borrow):
1. ONE idea per screen. Full-width blocks, each with a single headline, 1-2 lines of text, and exactly ONE button (MÁS INFORMACIÓN / DESCÁRGALA AQUÍ / ACCEDE AQUÍ). The scroll is a guided walk, not a wall.
2. Hero carousel up top: rotating announcements, one message at a time, one CTA.
3. "Servicios" card row right after the hero: icon + short title, task-oriented entry points.
4. Named sections with clear headers (NOTICIAS, SISTEMA ELECTRÓNICO), each = intro line + 2-3 supporting cards.
5. Sticky nav with search always visible; one prominent CTA button in the header.
6. Generous white space; calm two-color palette; alternating image/text sides.

PSRD translation (kid-friendly, per Kelvin: "sequential learning once the user selects a specific area"):
- Hero: rotating "¿Sabías que...?" facts from our real data, each deep-linking into its section.
- Journey entries: home cards become "¿Qué quieres aprender hoy?" with a suggested order.
- Inside each section: Paso 1→2→3 guided flow — (1) 30-second intro card, (2) explore the data, (3) "¿Y ahora qué?" footer linking the logical NEXT section. Chain: El país → Leyes → Sesiones → Provincias → Dinero.
- One idea per screen inside sections; secondary detail stays behind existing taps.
- Keep: all data, glossary, honesty notes, section color tokens, accessibility patterns.
